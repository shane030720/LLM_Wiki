## ADR-001. 프론트엔드: Next.js → 단일 HTML 파일

**배경**  
초기 README는 Next.js 14 프론트엔드를 계획했다.

**선택지**
- A. Next.js
- B. 단일 HTML 파일 (wiki_gui.html), FastAPI에서 직접 서빙

**결정: B**

**이유**
- 학생 개인 노트북에서 실행하는 로컬 도구이므로 빌드 파이프라인 오버헤드 불필요
- marked.js CDN 하나로 마크다운 렌더링이 충분
- Node.js 빌드 환경 없이 Python 환경만으로 전체 실행 가능

## ADR-002. LLM 호출: Claude CLI 기본, LiteLLM API 선택

**배경**  
초기 구현은 `claude -p` CLI subprocess. LiteLLM으로 전환했으나 API 키가 없으면 작동하지 않는 문제 발생. Claude Pro 구독자는 API 키 없이 CLI를 사용할 수 있는데 이를 활용할 방법 필요.

**최종 결정: CLI 방식과 API 방식 모두 지원**

```
LLM_MODEL=claude-cli              → claude -p subprocess (기본, API 키 불필요)
LLM_MODEL=gemini-cli              → gemini -p subprocess (무료 티어)
LLM_MODEL=anthropic/claude-...    → LiteLLM API (ANTHROPIC_API_KEY 필요)
LLM_MODEL=openai/gpt-4o          → LiteLLM API (OPENAI_API_KEY 필요)
LLM_MODEL=gemini/gemini-1.5-flash → LiteLLM API (GOOGLE_API_KEY 필요)
LLM_MODEL=ollama/llama3           → LiteLLM API (로컬, API 키 불필요)
```

**구현:** `llm_provider.py`가 `LLM_MODEL` 값으로 CLI/API 방식을 자동 선택. 기본값 `claude-cli`.

**이유**
- Claude Pro 구독자는 추가 비용 없이 즉시 사용 가능
- API 방식으로 OpenAI·Gemini·Ollama 등 다른 LLM도 지원
- `.env` 한 줄로 전환 — 에이전트 코드 변경 불필요

**배경**

- Python 네이티브, FastAPI와 자연스러운 통합

## ADR-006. Quiz 정답 숨김: QUIZ_ANSWER 마커 방식

**배경**  
문제를 읽다가 정답이 그냥 읽히는 문제 발생.

**선택지**
- A. 정답을 생성하지 않고 채점 시 가르쳐줌
- B. 문제/정답을 별도 API로 분리해 두 번 호출
- C. `QUIZ_ANSWER:` / `QUIZ_EXPLAIN:` 마커 출력 → 프론트엔드가 `<details>` 아코디언으로 변환

- JS 정규식 치환으로 구현 단순

## ADR-010. 클라우드 동기화: rclone 기반 multi-provider

**배경**  
OneDrive만 지원하다 Google Drive 추가 요청.

**결정: rclone 공통 인터페이스 + provider 선택**

- `onedrive.py` / `gdrive.py` — `list_folders()`, `list_files()`, `download_file()` 동일 인터페이스
- `.env`의 `DRIVE_PROVIDER=onedrive|gdrive` 로 기본 provider 선택
- `POST /api/sync/browse` + `POST /api/sync/pull` 로 UI 폴더 선택 동기화 지원

## ADR-011. Drive 폴더 선택 UI

**배경**  
Drive 전체가 아니라 강의자료 폴더만 `raw/`에 가져오고 싶다.

**결정: 브라우저 내 Drive 탐색 모달**

- `GET /api/sync/browse?provider=&path=` — rclone lsf로 하위 폴더 목록 반환
- `POST /api/sync/pull` — 선택한 폴더 → `raw/{subject}/` 동기화 + ChromaDB 임베딩
- UI: 폴더 클릭/더블클릭 탐색, 과목명 자동 채우기

## ADR-012. AgentMEMO 연동 방식

**배경**  
Back Agent가 무슨 작업을 하는지 사람이 볼 방법이 없었다.

**선택지**
- A. AgentMEMO HTTP MCP 서버(포트 8765)를 별도 프로세스로 실행
- B. LLM Wiki 백엔드에 `/agentmemo` 경로 추가, SQLite 직접 읽기

**결정: B**

**이유**
- 별도 포트·프로세스 없이 기존 서버에서 처리
- Back Agent가 SQLite에 직접 기록 → `/agentmemo` 뷰어가 2초마다 폴링
- AgentMEMO 뷰어에서 Ingest/점검 버튼으로 에이전트 직접 트리거 가능

## ADR-013. 다학기 운영: 인스턴스 복사 방식

**배경**  
LLM Wiki는 한 학기 분량이 한 서버의 실용적 상한. 다음 학기 운영 방법 결정 필요.

**선택지**
- A. 서버 인스턴스 복사 (포트 다르게)
- B. 단일 서버에 학기 레이어 추가
- C. 학기마다 raw/ 덮어씀

**결정: A + new-semester.sh 스크립트**

**이유**
- B는 코드 복잡도 증가 대비 실익 없음
- C는 이전 학기 접근 불가
- `bash new-semester.sh 8001` 한 줄로 즉시 실행, 기존 학기 보존

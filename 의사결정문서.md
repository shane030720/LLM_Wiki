## ADR-001. 프론트엔드: Next.js → 단일 HTML 파일

**배경**  
초기 README는 Next.js 14 프론트엔드를 계획했다.

**선택지**
- A. Next.js (계획안)
- B. 단일 HTML 파일 (wiki_gui.html), FastAPI에서 직접 서빙

**결정: B**

**이유**
- 학생 개인 노트북에서 실행하는 로컬 도구이므로 빌드 파이프라인 오버헤드 불필요
- marked.js CDN 하나로 마크다운 렌더링이 충분
- Node.js 빌드 환경 없이 Python 환경만으로 전체 실행 가능

## ADR-002. LLM 호출: Claude CLI → LiteLLM API

**배경**  
초기 구현은 `claude -p <prompt>` CLI를 subprocess로 호출했다. Claude Code 세션 한도에 걸리고, 다른 사람이 쓰려면 Claude 구독이 필수인 문제가 있었다.

**선택지**
- A. `anthropic` Python SDK 직접 사용
- B. `claude -p` CLI subprocess (초기 구현)
- C. LiteLLM — 100+ LLM을 단일 인터페이스로 추상화

**결정: C (B→C 전환)**

**이유**
- `.env`의 `LLM_MODEL` 한 줄로 Claude/OpenAI/Gemini/Ollama 전환 가능
- 특정 LLM 공급자에 종속되지 않음 — 다른 사람도 자신의 LLM으로 사용 가능
- `llm_provider.py` 단일 파일에 `run()` / `stream()` / `astream()` 통일 인터페이스
- Gemini 무료 티어, 로컬 Ollama 등 비용 없는 선택지 포함

**선택지**
- A. systemd 사용자 서비스 (`--user`)

## ADR-006. Quiz 정답 숨김: QUIZ_ANSWER 마커 방식

**배경**  
문제를 읽다가 정답이 그냥 읽히는 문제 발생.

**선택지**
- A. 정답을 생성하지 않고 채점 시 가르쳐줌
- B. 문제/정답을 별도 API로 분리해 두 번 호출
- C. 에이전트가 `QUIZ_ANSWER:` / `QUIZ_EXPLAIN:` 마커 출력 → 프론트엔드가 `<details>` 아코디언으로 변환

**결정: C**

**이유**

- 정답 없이 채점만 하면 학생이 스스로 확인 불가 (A 제외)
- JS 정규식 치환으로 구현 단순

## ADR-010. 클라우드 동기화: rclone 기반 multi-provider

**배경**  
OneDrive만 지원하다 Google Drive 추가 요청.

**결정: rclone 기반 공통 인터페이스**

```python
class DriveProvider:
    list_folders(path) → list[dict]
    list_files(path)   → list[dict]
    download_file(src, dest)
    load_hash_store() / save_hash_store()
```

- `onedrive.py` / `gdrive.py` — 동일 인터페이스, `RCLONE_REMOTE`만 다름
- `.env`의 `DRIVE_PROVIDER=onedrive|gdrive` 로 기본 provider 선택
- POST `/api/sync` — provider 파라미터로 선택적 동기화

## ADR-011. Drive 폴더 선택 UI

**배경**  
Drive 전체가 아니라 강의자료가 담긴 특정 폴더만 `raw/`에 가져오고 싶다.

**결정: 브라우저 내 Drive 탐색 모달**

- GET `/api/sync/browse?provider=&path=` — rclone lsf로 하위 폴더 목록 반환
- POST `/api/sync/pull` — 선택한 폴더 → `raw/{subject}/` 동기화 + ChromaDB 임베딩
- UI: 클릭으로 폴더 선택, 더블클릭/→으로 하위 탐색, 과목명 자동 채우기

**이유**
- Drive 전체를 한 번에 내려받으면 강의자료 외 파일도 섞임
- 학생이 원하는 폴더만 골라서 특정 과목으로 매핑 가능

## ADR-012. AgentMEMO 연동 방식

**배경**  
Back Agent가 무슨 작업을 하는지 사람이 볼 방법이 없었다.

**선택지**
- A. AgentMEMO MCP 서버(포트 8765)를 별도 프로세스로 실행
- B. LLM Wiki 백엔드에 `/agentmemo` 경로를 추가, SQLite 직접 읽기

**결정: B**

**이유**
- 별도 포트·프로세스 없이 기존 서버에서 처리
- AgentMEMO MCP 서버(HTTP Streamable)는 AI 에이전트 연동용 — 사람 뷰어에는 불필요
- Back Agent가 SQLite에 직접 기록 → `/agentmemo` 뷰어가 2초마다 폴링
- AgentMEMO 뷰어에서 Ingest/점검 버튼으로 에이전트 직접 트리거 가능

## ADR-013. 다학기 운영: 인스턴스 복사 방식

**배경**  
LLM Wiki는 1학기 분량이 한 서버의 실용적인 상한. 다음 학기에 어떻게 할지 결정 필요.

**선택지**
- A. 서버 인스턴스 복사 (포트 다르게)
- B. 단일 서버에 학기 레이어 추가
- C. 학기마다 raw/ 덮어씀

**결정: A + new-semester.sh 스크립트**

**이유**
- B는 코드 복잡도 증가 대비 실익 없음 (1학기 데이터가 충분히 독립적)
- C는 이전 학기 접근 불가
- A는 `bash new-semester.sh 8001` 한 줄로 즉시 실행
- 기존 학기는 그대로 보존, API 키 자동 재사용

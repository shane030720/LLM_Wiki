## ADR-001. 프론트엔드: Next.js → 단일 HTML 파일

**배경**  
초기 README는 Next.js 14 프론트엔드를 계획했다.

**선택지**
- A. Next.js (계획안)
- B. 단일 HTML 파일 (wiki_gui.html), FastAPI에서 직접 서빙

**결정: B**

**이유**
- 학생 개인 노트북에서 실행하는 로컬 도구이므로 빌드 파이프라인 오버헤드 불필요
- FastAPI의 `FileResponse`로 단일 파일 서빙 시 별도 웹서버·CDN 불필요
- marked.js CDN 하나로 마크다운 렌더링이 충분
- Node.js 빌드 환경 없이 Python 환경만으로 전체 실행 가능

## ADR-002. LLM 호출: API 직접 호출 → Claude Code CLI subprocess

**배경**  
에이전트가 Claude에 요청을 보내는 방법 결정 필요.

**선택지**
- A. `anthropic` Python SDK로 직접 API 호출
- B. `claude -p <prompt>` CLI를 subprocess로 호출

**결정: B**

**이유**
- 개발 환경에 Claude Code CLI가 이미 설치·인증되어 있음
- SDK 사용 시 API 키 관리·환경변수 별도 설정 필요
- CLI는 스트리밍(`--output-format stream-json`)을 내장 지원
- 에이전트 코드가 프롬프트 엔지니어링에 집중 가능, 인프라 코드 최소화
- 단점: Claude Code 세션 한도에 묶임, 병렬 처리 시 제약

## ADR-003. 벡터 DB: ChromaDB 선택

**배경**  
RAG 파이프라인에 사용할 벡터 스토어 선택 필요.

**선택지**
- A. ChromaDB (로컬 퍼시스턴트)
- B. Pinecone (클라우드)
- C. FAISS (인메모리)

**결정: A**

**이유**
- 로컬 퍼시스턴트로 서버 재시작 후에도 인덱스 유지 (FAISS 제외 이유)
- 추가 인프라·비용·인터넷 연결 불필요 (Pinecone 제외 이유)
- Python 네이티브, FastAPI와 자연스러운 통합
- 현재 데이터 규모(28개 파일, 수천 청크)에서 성능 충분

## ADR-004. 임베딩 모델: paraphrase-multilingual-MiniLM-L12-v2

**배경**  
한국어 강의자료를 임베딩할 모델 선택 필요.

**선택지**
- A. `paraphrase-multilingual-MiniLM-L12-v2` (sentence-transformers, 로컬)
- B. OpenAI `text-embedding-3-small` (API)
- C. Voyage AI `voyage-2` (API)

**결정: A**

**이유**
- 한국어·영어 혼용 텍스트를 단일 모델로 처리 가능 (multilingual)
- 로컬 실행이므로 API 비용·지연 없음
- 384차원으로 가볍고 ChromaDB와 빠른 코사인 검색 가능
- 오프라인 환경에서도 동작

## ADR-005. 서비스 관리: systemd 사용자 서비스

**배경**  
서버를 상시 실행하고 재부팅 후 자동 복구 방법 결정 필요.

**선택지**
- A. systemd 사용자 서비스 (`--user`)
- B. Docker Compose
- C. 수동 실행 (`python` 명령어)

**결정: A**

**이유**
- WSL2 환경에서 Docker 데몬 오버헤드 불필요
- `systemctl --user`로 root 권한 없이 관리 가능
- 부팅 시 자동 시작, 크래시 시 자동 재시작
- `journalctl --user -u llm-wiki.service`로 로그 중앙 관리

## ADR-006. Quiz UX: 정답 아코디언 숨김 처리

**배경**  
퀴즈 문제와 정답이 같이 표시되어 문제를 읽다가 정답이 노출되는 문제 발생.

**선택지**
- A. 정답을 아예 생성하지 않고 채점 시 답을 가르쳐줌
- B. 문제와 정답을 별도 API로 분리해 두 번 호출
- C. 에이전트가 특수 마커(`QUIZ_ANSWER:`, `QUIZ_EXPLAIN:`)를 출력, 프론트엔드에서 `<details>` 아코디언으로 변환

**결정: C**

**이유**
- API 추가 호출 없이 단일 응답으로 처리 가능 (B 제외 이유)
- 정답 없이 채점만 하면 학생이 정답을 스스로 확인할 방법이 없음 (A 제외 이유)
- 마커 파싱은 JS 정규식으로 간단하게 구현
- 정답을 클릭 전까지 완전히 숨길 수 있음

## ADR-007. 에이전트 역할 경계: 시스템 프롬프트 방식

**배경**  
Quiz Agent에 위키 개념 질문을 해도 Quiz Agent가 그냥 답변하는 문제. 두 에이전트가 역할 구분 없이 동작.

**선택지**
- A. 시스템 프롬프트에 역할 경계 명시 (A 방안)
- B. 퀴즈탭에 별도 대화창 추가 + 백엔드 의도 분류 라우팅 (B 방안)
- C. 퀴즈탭 UI를 버튼 방식으로만 제한

**결정: A**

**이유**
- B는 대화창을 추가해도 결국 같은 모델이 답변하므로 구조적 차이가 없음
- C는 "서술형으로 내줘", "다른 문제 내줘" 등의 유용한 conversational 기능이 사라짐
- A는 코드 변경 최소로 즉시 적용 가능
- 프롬프트 첫 번째 섹션에 역할 경계를 명시해 모델이 우선적으로 처리하도록 함
- 단점: LLM이 확률적으로 규칙을 무시할 수 있음 (허용 가능한 수준)

## ADR-008. 다크 테마 텍스트 색상 기준

**배경**  
배경색(`#1a1b26`)과 유사한 텍스트 색상(`#4a5078`, `#6b7699`)이 광범위하게 사용되어 가독성 문제 발생.

**결정**  
텍스트 색상 3단계 체계 수립:

| 용도 | 색상 | 적용 예시 |
|------|------|-----------|
| 주요 텍스트 | `#c0caf5` | 제목, 활성 탭 |
| 보조 텍스트 | `#a9b1d6` | 본문, 파일명 |
| 3차 텍스트 (메타) | `#8892b0` | 부제목, 레이블, 탭 비활성 |
| 약한 텍스트 (힌트) | `#7c88b0` | 플레이스홀더, 화살표, 상태 표시 |

`#4a5078`은 텍스트 용도 완전 제거 (border/background에만 허용).

## ADR-009. Quiz Agent 출력에서 blockquote 금지

**배경**  
Claude가 출처를 `> 출처: ...` blockquote 형식으로 출력하면, 다크 배경의 `.md blockquote { color: #6b7280 }` 스타일이 적용되어 텍스트가 배경에 묻힘.

**결정**  
시스템 프롬프트에 "블록인용(>) 문법 절대 사용 금지" 규칙을 추가하고, 출처는 `**출처:**` bold 형식으로 대체.

추가로 CSS에서 `.quiz-output .md` 하위 모든 요소에 밝은 색 강제 override (`p`, `li`, `h1`, `h2`, `blockquote`, `table` 등 전체 커버).

## ADR-010. Back Agent 자동 실행 시점

**배경**  
Back Agent를 언제 실행할지 결정 필요.

**선택지**
- A. 서버 시작(`lifespan`) 시 1회 자동 실행
- B. 주기적 크론잡 (예: 매일 자정)
- C. 사용자가 수동으로 실행

**결정: A + C 병행**

**이유**
- 서버 재시작 = 보통 새 파일 추가 후이므로 시작 시 점검이 자연스러움
- `asyncio.create_task(_run_back_agent())`로 메인 서버 기동을 블로킹하지 않음
- 수동 실행(`python back_agent.py`)도 언제든 가능
- 현재 규모에서 주기적 실행은 과도함

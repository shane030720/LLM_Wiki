## Definition

Classful Addressing(클래스풀 주소 지정)은 초기 인터넷에서 사용된 IPv4 주소 체계로, 32비트 주소를 고정된 클래스(A, B, C, D, E)로 분류하고 클래스별로 네트워크 식별자(netid)와 호스트 식별자(hostid)를 고정 경계로 분리하는 방식이다. 각 인터넷 조직은 하나 이상의 IP 주소 블록(= 하나의 netid)을 할당받았으며, 같은 netid 내 호스트들은 동일 네트워크에 속했다.

## How It Works

### 클래스 분류 (최상위 비트 패턴 기준)

| 클래스 | 시작 비트 패턴 | 주소 범위 | netid 크기 | hostid 크기 | 최대 네트워크 수 | 네트워크당 최대 호스트 수 |
|--------|--------------|-----------|-----------|------------|----------------|------------------------|
| A | `0` | 0.0.0.0 ~ 127.255.255.255 | 8 bits | 24 bits | 128 | 16,777,214 |
| B | `10` | 128.0.0.0 ~ 191.255.255.255 | 16 bits | 16 bits | 16,384 | 65,534 |
| C | `110` | 192.0.0.0 ~ 223.255.255.255 | 24 bits | 8 bits | 2,097,152 | 254 |
| D | `1110` | 224.0.0.0 ~ 239.255.255.255 | 멀티캐스트 주소 | — | — | — |
| E | `1111` | 240.0.0.0 ~ 255.255.255.255 | 예약됨 | — | — | — |

### 라우팅 방식

1. 수신 데이터그램의 목적지 IP 주소에서 상위 비트를 확인하여 클래스 판별
2. 클래스별 고정 네트워크 마스크를 IP 주소에 AND 연산 → 네트워크 주소 추출
   - Class A: 255.0.0.0 (`/8`)
   - Class B: 255.255.0.0 (`/16`)
   - Class C: 255.255.255.0 (`/24`)
3. 추출된 네트워크 주소로 라우팅 테이블 조회 → 출력 인터페이스 결정

### IPv4 주소 표기

32비트 주소를 8비트씩 4개 그룹으로 나누어 점 십진수(dotted-decimal) 표기:
- 예: `10100101 11110110 00101011 10001111` → `165.246.43.143`
- 각 십진수 범위: 0~255
- 유효하지 않은 표기 예: 선행 0(`045`), 4개 초과 바이트, 255 초과 값, 이진수·십진수 혼용

16진수 표기도 가능: `165.246.43.143` → `0xA5F62B8F`

### 한계 및 주소 낭비

- **Class A 낭비**: 하나의 블록에 최대 1600만 개 호스트를 수용하지만, 그만큼 큰 조직은 극소수
- **Class B 낭비**: 65,534개 호스트 블록도 대부분의 조직에게 과잉
- **Class C 부족**: 254개 호스트만 지원하므로 중간 규모 조직에게는 너무 작음
- **불공정 분배**: 대형 기관(MIT, DoD 등)이 Class A 전체 블록을 보유 (예: MIT는 `/8` 블록 보유)
- 결과적으로 IPv4 주소 공간이 조기에 소진되어 현재는 희소 자원이 됨

## Key Properties

- 주소의 상위 비트만 보면 클래스를 즉시 판별할 수 있는 자기 기술적(self-identifying) 구조
- netid와 hostid의 경계가 클래스에 따라 바이트 경계에서 고정됨
- 서브네팅(RFC 950) 도입 전까지는 클래스풀 주소만 존재했음 (인터넷 초기 1980년대)
- 1990년대 CIDR(Classless Inter-Domain Routing) 도입으로 대체됨
- 현재 IPv4 주소는 거래·임대되는 희소 자원 (Class A 블록 단가 지속 상승)

## Definition

Subnet Addressing(서브넷 주소 지정, RFC 950)은 1980년대 초 LAN의 급증에 대응하여 도입된 메커니즘으로, 기존 클래스풀 IP 주소 블록을 외부 인터넷 라우팅 인프라 변경 없이 조직 내부적으로 여러 서브네트워크(subnet)로 분할하는 방식이다. 기존 hostid 부분을 subnetid와 실제 hostid로 추가 분리하여 계층 구조를 만든다.

## How It Works

### 주소 구조 변화

```
클래스풀:  [ netid     |         hostid            ]
서브넷:    [ netid     | subnetid |    hostid       ]
```

- **netid**: 클래스에서 정의된 네트워크 식별자 (외부 인터넷에서 동일하게 인식)
- **subnetid**: 조직 내부에서 사용하는 서브네트워크 식별자
- **hostid**: 서브넷 내 개별 호스트 식별자

### 서브넷 마스크 (Subnet Mask)

호스트나 라우터가 IP 주소에서 (서브)네트워크 부분을 추출하기 위해 사용하는 비트 패턴:

```
서브넷 주소 = IP 주소 AND 서브넷 마스크
```

표현 방식:
1. **점 십진수 표기**: `255.255.255.0`
2. **프리픽스 길이(CIDR notation)**: `/24` (현재 더 일반적으로 사용)

**예시:** 사이트 전체 서브넷 마스크 `255.255.255.0` (/24) 적용 시:
- Class B 주소 블록을 256개의 서브네트워크로 분할
- 각 서브네트워크에 254개 호스트 수용 (0=네트워크 주소, 255=브로드캐스트 주소 제외)

### 서브넷 마스크 계산

서브넷 마스크에서:
- **1 비트 부분**: 네트워크/서브넷 식별 비트 (netid + subnetid)
- **0 비트 부분**: 호스트 식별 비트 (hostid)

비트 AND 연산을 통해 네트워크 주소 추출:
```
IP:   192.168.1.130  →  11000000.10101000.00000001.10000010
Mask: 255.255.255.0  →  11111111.11111111.11111111.00000000
결과: 192.168.1.0    →  11000000.10101000.00000001.00000000
```

### 라우팅 방식

클래스풀 라우팅과 동일하지만 클래스별 고정 네트워크 마스크 대신 **서브넷 마스크**를 사용:
1. 목적지 IP 주소 AND 서브넷 마스크 → 서브넷 주소 추출
2. 서브넷 주소로 포워딩 테이블 조회 → 출력 인터페이스 결정

### 서브넷 시각화

```
223.1.1.1, .2, .3, .4 → 서브넷 223.1.1.0/24
223.1.2.1, .2, .9     → 서브넷 223.1.2.0/24
223.1.3.1, .2, .27    → 서브넷 223.1.3.0/24
```

**서브넷 정의 방법 (conceptual recipe):**
- 각 인터페이스를 호스트·라우터에서 분리 → 고립된 네트워크 "섬" 생성
- 각 섬이 하나의 서브넷에 해당
- 동일 서브넷의 장치들은 라우터를 거치지 않고 물리적으로 직접 통신 가능

라우터 인터페이스 간 링크 (포인트-투-포인트) 역시 각각의 /30 또는 /31 서브넷을 구성한다.

## Key Properties

- 서브네팅은 외부 라우팅 인프라 변경 없이 조직 내부 네트워크를 분할
- 서브넷 마스크의 경계는 클래스풀 방식과 달리 비트 단위로 임의 설정 가능 (가변 길이 서브넷 마스크, VLSM)
- 동일 서브넷 장치들은 라우터 없이 직접 통신; 서브넷 간 통신은 라우터 경유
- 프리픽스 길이 표기(`/24` 등)가 현재 표준이며, 서브넷 마스크 점 십진수 표기보다 간결
- RFC에서는 원문 기준 subnetid 부분을 "subnet id"라고 표현 (교재에서는 "subnetid")

## Open Questions

- 가변 길이 서브넷 마스크(VLSM)를 사용하면 주소 공간을 클래스풀 대비 얼마나 효율적으로 활용할 수 있는가?
- 서브넷 주소 체계와 CIDR은 개념적으로 어떻게 구분되는가? CIDR이 서브넷 개념을 완전히 포함하는가?
- 라우터 인터페이스 링크에도 서브넷 주소를 할당하는 것이 항상 필요한가? (unnumbered interface 방식과의 비교)

# Domain — LLM Wiki 지식 도메인 정의

이 문서는 LLM Wiki가 다루는 지식의 범위, 자료 구조, 처리 방식을 정의한다.
에이전트는 이 범위를 벗어난 내용을 wiki에 추가하거나 답변에 포함해서는 안 된다.

---

## 1. 지식 도메인 — 6개 과목

| 과목 | raw/ 폴더 | 유형 |
|------|-----------|------|
| 자료구조 | `raw/자료구조/` | 이론 + 알고리즘 |
| 알고리즘 | `raw/알고리즘/` | 이론 + 알고리즘 |
| 데이터베이스 | `raw/데이터베이스/` | 이론 + SQL |
| 컴퓨터네트워크 | `raw/컴퓨터네트워크/` | 이론 + 프로토콜 |
| 시스템 분석 이론 | `raw/시스템 분석 이론/` | 방법론 + 모델링 |
| 시스템 분석 실습 | `raw/시스템 분석 실습/` | 실습 자료 + 코드 |

각 과목 폴더 아래에 강의 슬라이드 PDF를 넣으면 Edit Agent가 자동으로 wiki 페이지로 변환한다.

---

## 2. 자료 투입 방식

### 로컬 파일

```
raw/
└── {과목명}/
    └── {파일명}.pdf     ← 여기에 강의자료 배치
```

### Google Drive 연동

웹 UI 우측 상단 **☁️ Drive** 버튼으로 연결한 Google Drive 폴더와 동기화할 수 있다.
Drive에서 파일을 가져오면 해당 `raw/{과목}/` 경로에 자동 복사된 뒤 ingest 파이프라인으로 전달된다.

---

## 3. 자료 유형 제약

| 허용 | 제외 |
|------|------|
| 교수 제공 강의 슬라이드 PDF | 손글씨 필기 내용 |
| 텍스트 레이어가 있는 디지털 PDF | 이미지 전용 스캔본 (OCR 없음) |
| 별표(★/☆/*) 표시 중요 내용 | "시험 안나옴" 표시 내용 |

---

## 4. 필기 PDF 처리 방식

Edit Agent는 `pypdf`로 PDF의 디지털 텍스트 레이어만 추출한다. 손글씨는 추출되지 않는다.
손글씨가 포함된 스캔 PDF는 텍스트가 없는 것으로 판정되어 자동으로 `skipped` 처리된다.

### 중요도 판별 규칙

| 표시 | 처리 |
|------|------|
| 별표(★, ☆, *) | 해당 내용을 `[중요]` 태그와 함께 더 상세하게 정리 |
| "시험 안나옴" / "시험 X" | 해당 내용 정리 제외 |
| 표시 없음 | 핵심만 간략하게 정리 |

---

## 5. Wiki 페이지 분류 체계

| 분류 | 폴더 | 의미 | 예시 |
|------|------|------|------|
| concept | `wiki/pages/concepts/` | 개념·이론 정의 (what a thing IS) | Binary Search Tree, TCP 3-way Handshake |
| entity | `wiki/pages/entities/` | 도구·시스템·알고리즘 (who/what EXISTS) | Dijkstra Algorithm, B+ Tree |
| synthesis | `wiki/pages/syntheses/` | 비교·분석·교차 출처 (cross-source analysis) | 정렬 알고리즘 비교, OSI vs TCP/IP |

모든 페이지는 `LLM.md` 스키마에 정의된 frontmatter와 섹션 구조를 따른다.

---

## 6. 도메인 외 처리 원칙

- Edit Agent는 강의자료 원본에 없는 내용을 wiki에 추가하지 않는다.
- Wiki Agent는 wiki + 강의 노트에 없는 내용을 질문받으면 DuckDuckGo 웹검색(API 키 불필요)으로 보완한다.
- Quiz Agent는 Edit Agent가 정리한 내용 기반으로만 문제를 출제한다. 자료에 없는 개념으로 문제를 만들지 않는다.
